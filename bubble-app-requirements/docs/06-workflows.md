## Bubble ワークフローとは

Bubble のワークフローは、ユーザーのアクションやイベントに応じて自動実行される処理のことです。
「When（いつ）」→「Do（何をする）」の形式で定義します。

## ワークフロー詳細

### WF-01: ログイン処理

**When:** 「ログイン」ボタンがクリックされた時

**条件:**
- Input Email の値が空でない
- Input Password の値が空でない

**Steps:**
1. `Log the user in` (Email: Input Email, Password: Input Password)
2. `Navigate to` > Dashboard ページ

**エラー時:**
- ログイン失敗の場合: エラーメッセージを表示する

---

### WF-02: 会員登録処理

**When:** 「登録する」ボタンがクリックされた時

**条件:**
- 各入力項目が空でない
- パスワードが一致している
- 利用規約に同意している

**Steps:**
1. `Sign the user up` (Email, Password, 追加フィールド)
2. `Send email` (確認メール送信)
3. `Navigate to` > Dashboard ページ

---

### WF-03: 認証チェック

**When:** ページが読み込まれた時

**条件:** Current User is not logged in

**Steps:**
1. `Navigate to` > ログインページ

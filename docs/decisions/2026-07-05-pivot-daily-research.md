# 判断記録: プラス化に向けた日足×長期リサーチへのピボット（2026-07-05・人間承認済み）

> 1行サマリー: エッジ分析（`2026-07-05-edge-analysis.md`）の結論を受け、オーナー承認「A、もしくは全くの別案」に基づき、(H1)日足トレンドフォロー×6年データ と (H2)fundingキャリー[別案・構造的エッジ] と (H3)リバランス・プレミアム[別案] の3仮説を検証するリサーチ基盤を追加。**正のエッジが実証された手法のみをbot化する**。

## 承認内容と方針

- 現行の1時間足以下の世界にエッジが無いことは実証済み。以後、**エッジの実証 → bot化**の順番を厳守する（逆にしない）。
- 検証で全仮説が棄却された場合は「この条件でプラス化は不可」を結論として報告する（それも成果とする）。

## 検証する3仮説

| # | 仮説 | エッジの源泉 | データ |
|---|---|---|---|
| H1 | 日足トレンドフォロー（SMAクロス50/200・20/100、Donchian55/20、TSMOM90d） | 行動バイアス由来のトレンド持続。年数回転でコスト影響が2桁小 | Binance日足 2020〜（強気・弱気サイクル込み約6.5年） |
| H2 | fundingキャリー（現物ロング＋perpショート） | **構造的**: perpのロング偏りが支払う資金調達率を受け取る。方向予測不要・市場中立 | Binance USDM funding履歴 2020〜 |
| H3 | 50/50リバランス・プレミアム | ボラティリティ・ハーベスティング（高ボラ資産の凹性利得） | H1と同一 |

共通条件: コスト0.15%/片道（キャリーは全4レッグ）、シグナルは終値確定→翌日反映（ルックアヘッド防止）、IS（〜2024）/OOS（2025〜）分割で過剰適合を検出。

## 追加したもの

- `scripts/fetch_research_data.py`: Binance/Kraken日足＋funding履歴を `data/research.db` へ取得
- `.github/workflows/research_data.yml`: 上記を実行する手動ワークフロー（workflow_dispatch）
- `scripts/backtest_daily.py`: 3仮説の検証スイート（総リターン・CAGR・最大DD・Sharpe・取引数・建玉率）

## 実行手順（このサンドボックスは外部API到達不可のため）

1. PR #1 をmainへマージ
2. `research_data` ワークフローを実行（司令塔がGitHub API経由で起動可能）
3. `data/research.db` がmainにコミットされたら `python scripts/backtest_daily.py` で判定
4. 結果と「bot化する/しない」の判断を `docs/decisions/` に記録

## 判定基準（事前に固定・後出し禁止）

bot化に進む条件（銘柄・戦略ごと）:
- OOS（2025〜）で **CAGRが正** かつ **Buy&HoldよりSharpeが高い** かつ **最大DDがB&Hより浅い**
- IS/OOSで成績の符号が一致（ISだけ好成績はカーブフィットとみなし不採用)
- H2は「funding受取の年率がコスト・想定執行誤差(0.2%/年)を控除して正」

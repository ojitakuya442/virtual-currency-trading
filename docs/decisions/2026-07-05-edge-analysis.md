# 分析記録: 「プラスにするには何が必要か」のエッジ検証（2026-07-05）

> 1行サマリー: オーナーの要求「損失縮小ではなくプラス化」を受けた追加検証の結論 — **現有戦略にもレジームスイッチにも、1時間足×コスト0.15%の世界で正のエッジは存在しない**。上昇局面(+25%)でもbot群は-2.7%、レジームスイッチ単体は-13〜-19%。プラス化には時間軸の変更（日足×長期データでの再検証）か検証目的の再定義が必要。

## 検証1: 新構成bot群の局面別成績（クリーンデータ・129日）

| 局面 | B&H (BTC/ETH) | 6bot合計（新構成） |
|---|---|---|
| 上昇・レンジ (02-25〜05-06) | **+24.5% / +24.9%** | **-8,136円 (-2.71%)** |
| 暴落・反発 (05-06〜07-05) | -22.7% / -25.6% | -9,953円 (-3.32%) |
| 全期間 | -3.7% / -7.1% | -17,441円 (-5.81%) |

**決定的な事実**: +25%の追い風でもbot群はマイナス。上昇局面のグロスは+5,514円と正だが、フェーズ1適用後でもコスト13,650円がそれを上回る。**問題はコストだけでなく、シグナル自体のエッジが微弱**（トレンド捕捉率 約7%）。

## 結論（コールドリーディング）

1. この4.5ヶ月・1時間足・片道0.15%の条件下で、**検証した全手法が「取引しない（現金=0%）」に負けている**。
2. コスト削減（フェーズ1）は出血を半減させたが、エッジが無い以上プラスには到達しない。ML（メタラベリング）も**増幅すべきエッジが存在しなければ機能しない**ため、現時点では順序が違う。
3. プラス化の現実的な候補は時間軸の引き上げ: トレンドフォローが実績を持つのは**日足×複数年**の世界（保有数週間〜数ヶ月、年数回転＝コスト影響が2桁小さい）。現DBには139日分しかなく検証不能。Kraken公開APIから2〜3年の日足を取得すれば、bot再設計の前に「そこにエッジがあるか」をデータで判定できる。
4. 対価も明示する: 日足化すると取引は月数回になり、15分間隔で動かす現行インフラの意味は薄れる（検証プロジェクトとしての性格が変わる）。

## 選択肢（人間の判断待ち）

- **B. 現状維持で観察**: フェーズ1構成のまま7/19評価まで走らせ、実運用データで判断。
- **C. 検証終了**: 「この条件では技術的取引に正のエッジなし」を本プロジェクトの結論として撤収。
- （非推奨）D. 現時間軸のままML着手: エッジ不在のため期待値が立たない。
